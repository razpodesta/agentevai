1. VISÃO ESTRATÉGICA DO ORGANISMO
O ecossistema Agentevai evoluiu de uma aplicação modular para uma Infraestrutura de Soberania Digital. Nossa arquitetura não aceita "ajustes"; ela opera via Reconfiguração Genética. O objetivo é transformar o dado bruto (o petróleo social) em verdade matemática inalterável através de uma refinaria de inteligência artificial e blockchain.
2. JUSTIFICATIVA VERBOSA DOS NOVOS WORKSPACES (A PERÍCIA DO ELITE ARCHITECT)
Abaixo, apresento a fundamentação lógica para cada búnquer criado, justificando por que esta é a única decisão correta para um software de classe mundial:
2.1. @agentevai/registry-vault (O Cartório de Dados)
Decisão Lógica: Isolamento total do "Ser" (Identidade) do "Poder" (Autorização).
Justificativa: Em arquiteturas tradicionais, o usuário e suas permissões vivem na mesma tabela. No Agentevai, isso é considerado uma falha de segurança e escalabilidade. O registry-vault é o único SSOT (Single Source of Truth) para dados biométricos, preferências de interface (Obsidiana/Milk), avatares e credenciais IAL.
Vantagem de Elite: Permite trocar a lógica de permissões sem nunca tocar nos dados sensíveis do cidadão, garantindo conformidade eterna com a LGPD.
2.2. @agentevai/sovereign-authorization (O Búnquer da Lei)
Decisão Lógica: Centralização da lógica de permissões em um motor de regras puro.
Justificativa: "Quem você é" não deve ditar automaticamente "o que você pode fazer" sem uma camada de intermediação legal. Este workspace é o Tribunal do sistema. Ele intercepta cada rastro e pergunta: "Este rastro tem autoridade para selar esta notícia neste hexágono H3?".
Vantagem de Elite: Erradica o espalhamento de if (user.role === 'ADMIN') por todo o código, centralizando a soberania em uma única aduana.
2.3. @agentevai/gamification-engine (A Doutrina de Mérito)
Decisão Lógica: Meritocracia social tratada como uma moeda matemática independente.
Justificativa: O Standing (Reputação) é o motor da confiança. Ao isolar este cálculo, permitimos que as regras de pontuação evoluam (ex: bônus para jornalistas, punição para bots) sem afetar o core da identidade.
Vantagem de Elite: Permite a implementação do IRS (Índice de Relevância Soberana), onde o peso da voz de um cidadão é um rastro dinâmico e auditável.
2.4. @agentevai/intelligence-hub (O Maestro Neural)
Decisão Lógica: Desacoplamento de provedores de IA e orquestração de custos.
Justificativa: Depender de uma única API (OpenAI ou Anthropic) é uma falha de soberania. O Hub negocia qual modelo é melhor para cada tarefa (Custo vs. Precisão).
Vantagem de Elite: Permite o "Handshake de Economia". Se a tarefa é simples (limpeza de texto), usa-se o Llama 3 (Groq/Free). Se é perícia forense de código, escala-se para o Claude 3.5.
2.5. @agentevai/data-refinery & @agentevai/neural-vault (A Refinaria e a Memória)
Decisão Lógica: Implementação de Memória Vetorial (RAG) para economia de tokens.
Justificativa: Dados são o petróleo. O data-refinery limpa a radiação emocional do relato do cidadão, e o neural-vault armazena o significado (embeddings).
Vantagem de Elite: Se 100 pessoas denunciam o mesmo problema, a IA "lembra" do primeiro e automatiza os outros 99 com custo zero de raciocínio.
2.6. @agentevai/editorial-bunker & @agentevai/synthetic-ingestion (A Fábrica da Verdade)
Decisão Lógica: Automação do ciclo de vida da notícia desde fontes externas até a selagem imutável.
Justificativa: Portais tradicionais dependem de editores humanos lentos. Nossa estrutura permite que a IA "naturalize" conteúdos externos e que o enxame popular (Swarm) gere notícias automáticas ao atingir quórum.
Vantagem de Elite: Garante que o Agentevai tenha o maior volume de informações locais do Brasil, mantendo o "Estilo Soberano" em cada rastro de texto.
2.7. @agentevai/sovereign-taxonomy (O Dicionário Ontológico)
Decisão Lógica: Padronização gramatical de todas as dores e responsabilidades brasileiras.
Justificativa: Erradica a alucinação da IA. O sistema não "chuta" categorias; ele seleciona identificadores na Universal Taxonomy Matrix (UTM).
Vantagem de Elite: Permite ruteamento B2B automático. Se a categoria é SEWAGE_LEAK, o sistema sabe instantaneamente qual CNPJ responde por aquele hexágono H3.
3. AS OITO LEIS DE EXECUÇÃO ZENITH (MANDATÓRIO)
Higiene Lexical Absoluta: Proibido o uso de id, req, res, params, props, t, err. Use identifier, request, response, parameters, properties, translate, error.
Sincronia de ADN Zod V4: Todo aparato deve ter seu schemas/ApparatusName.schema.ts. Use obrigatoriamente .brand<'Name'>() e .readonly().
Trindade Linguística: Nenhum workspace é aceito sem os silos i18n/pt-BR, es-ES, en-US sincronizados.
Resolução ESM NodeNext: Todas as importações relativas devem portar a extensão .js.
Aduana de Borda: Componentes pai devem validar o ADN dos filhos via parse() antes da injeção de propriedades.
Unificação de Rastro: O único identificador de jornada permitido é o correlationIdentifier.
Zero Any Policy: O uso de any é um crime de radiação técnica que invalida a soberania do código.
Internal Mirroring: Schemas e Locales das aplicações devem residir internamente para otimizar o build Zenith.
